### Overall Assessment

**1. Program Content**

The program is structured in a reasonable progression, starting with fundamental AWS knowledge before moving on to practical implementation. The combination of theory and hands-on practice gives learners the opportunity to apply what they have learned immediately. As a result, when they entered the team Workshop, the implementation of the assigned infrastructure tasks became much smoother.

**2. Quality of Materials**

The learning materials and Workshop instructions are presented in a fairly detailed manner, with illustrations and clear step-by-step guidance. Dividing the content by team members helps define the scope of work precisely and reduces overlap during implementation. In addition, AWS official documentation is also a valuable reference source for understanding each service more deeply.

**3. Practical Value of the Program**

One of the program’s strengths is that it does not only introduce AWS services individually, but also teaches how to combine them into a complete system. In the ZeroBug Agent Workshop, the implementation of IAM and VPC helped establish a shared foundation for the whole system and clarified the connections between services such as EC2, Lambda, API Gateway, CloudFront, and WAF.

**4. Skills Gained**

After 12 weeks of study and practice, knowledge of identity management with IAM, network infrastructure design using Amazon VPC, Public/Private Subnet, NAT Gateway, and Security Groups was significantly strengthened. At the same time, planning, teamwork, and infrastructure handover skills among team members also improved through the Workshop process.

**5. Challenges and Lessons Learned**

- **Biggest challenge:** Designing the network architecture and IAM permissions so that team members can share the infrastructure while still ensuring security and manageability.
- **Another challenge:** Synchronizing infrastructure parameters and handing them over accurately to the next team members to avoid affecting the implementation timeline.
- **Lesson learned:** Resource naming conventions should be standardized from the beginning, a clear implementation plan should be established, and communication between team members should be maintained regularly to reduce mistakes during execution.

**6. Program Support**

The AWS Credits provided by the program made it much easier to practice in a real AWS environment without worrying too much about cost. In addition, the Workshop sessions and instructional materials were highly supportive throughout the learning and project implementation process, and I received a great deal of support from the program admins and others.

### Additional Questions

- **What was the most satisfying part of participating in the program?**

  → The program admins were very dedicated in supporting me, giving thoughtful advice and clear explanations that were easy to understand. I was also able to directly implement AWS infrastructure for a complete system and work with team members to build the product through a real-world workflow.

- **What should the program improve?**

  → It should add more in-depth practical sessions on AWS architecture optimization, common troubleshooting, and sharing experiences from real projects.

- **Would you be willing to recommend the program to others? Why?**

  → Yes. The program is suitable for students who want to learn AWS in a structured way, combining theory and practice while also gaining experience through team projects and events.

---

### Suggestions and Expectations

- Add more Workshops on cost optimization, system monitoring, and security on the AWS platform.
- Increase the number of scenario-based labs so learners have more opportunities to practice.
- Organize more sharing sessions with mentors or experienced professionals who have implemented AWS projects in companies.
- Continue maintaining the learning model that combines group Workshops to improve both technical knowledge and teamwork skills.
