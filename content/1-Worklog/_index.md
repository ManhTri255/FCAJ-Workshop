**On this page** you will find the worklog recorded over **12 weeks** of the First Cloud Journey internship (Apr 20 – Jul 12, 2026). The process includes AWS Cloud Journey learning, knowledge review, project planning, and AWS infrastructure deployment for the **ZeroBug Agent** project.

**Week 1 (Apr 20 – Apr 26):** [Register and activate AWS account, learn Module 01 fundamentals of cloud computing, receive 100 USD AWS Credits](1.1-week1/)

**Week 2 (Apr 27 – May 3):** [Explore AWS global infrastructure, service management tools, practice cost management and optimization through AWS Budgets Labs](1.2-week2/)

**Week 3 (May 4 – May 10):** [Build VPC, create public and private Subnets, configure Internet Gateway, prepare Key Pair](1.3-week3/)

**Week 4 (May 11 – May 17):** [Configure NAT Gateway, routing Route Tables, create Security Groups for VPC access control](1.4-week4/)

**Week 5 (May 18 – May 24):** [Deploy EC2 instances in VPC, test connectivity between components, clean up resources](1.5-week5/)

**Week 6 (May 25 – May 31):** [Consolidate AWS knowledge learned, review services, apply to ZeroBug Agent project](1.6-week6/)

**Week 7 (Jun 1 – Jun 7):** [Analyze ZeroBug Agent system requirements, build AWS deployment architecture, plan and divide tasks](1.7-week7/)

**Week 8 (Jun 8 – Jun 14):** [Design AWS architecture diagram, database model ERD, Wireframe and system UI/UX interface](1.8-week8/)

**Week 9 (Jun 15 – Jun 21):** [Finalize architecture diagram based on feedback, research Route 53, CloudFront, and AWS WAF for Workshop](1.9-week9/)

**Week 10 (Jun 22 – Jun 28):** [Deploy two-tier VPC, EC2 in Private Subnet, Application Load Balancer, and Amazon SQS](1.10-week10/)

**Week 11 (Jun 29 – Jul 5):** [Integrate AWS infrastructure into team system, test processing workflow, adjust configuration for stability](1.11-week11/)

**Week 12 (Jul 6 – Jul 12):** [Finalize system, update documentation, consolidate implementation results, and prepare for acceptance phase](1.12-week12/)
